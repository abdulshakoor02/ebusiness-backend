# Lead Import API Enhancement Plan

## Overview
Enhance the lead import API to support two-step column mapping confirmation:
1. User uploads file → Backend returns column mapping preview
2. Frontend confirms/edits mapping → Backend proceeds with import

## Current Implementation
- **Endpoint:** `POST /leads/import`
- **Flow:** Upload file → Parse → Auto-map columns (AI/heuristic) → Process rows → Bulk insert
- **Key files:**
  - `internal/adapters/handler/lead_handler.go:201-242`
  - `internal/core/services/lead_service.go:220-432`
  - `pkg/excel/mapper.go` - Column mapping logic
  - `pkg/excel/parser.go` - File parsing

## API Flow

### Step 1: Preview
```
POST /leads/import/preview
Content-Type: multipart/form-data

file: <uploaded file>

Response:
{
  "session_id": "uuid-here",
  "expires_at": "2024-01-01T12:30:00Z",
  "headers": ["Name", "Email", "Phone", "Status", "Source"],
  "sample_rows": [["John Doe", "john@email.com", "+123456", "Hot Lead", "Website"], ...],
  "total_rows": 100,
  "suggested_mappings": [
    {"column_index": 0, "header_name": "Name", "target_field": "full_name", "confidence": 0.95},
    {"column_index": 1, "header_name": "Email", "target_field": "email", "confidence": 1.0},
    ...
  ],
  "available_target_fields": [...],
  "existing_categories": [{"id": "...", "name": "Hot Lead"}, ...],
  "existing_sources": [{"id": "...", "name": "Website"}, ...],
  "existing_qualifications": [...]
}
```

### Step 2: Confirm and Import
```
POST /leads/import/confirm
Content-Type: application/json

{
  "session_id": "uuid-here",
  "assigned_to": "<optional user id>",
  "mappings": [
    {"column_index": 0, "header_name": "Name", "target_field": "full_name"},
    {"column_index": 1, "header_name": "Email", "target_field": "email"},
    ...
  ]
}

Response:
{
  "total_rows": 100,
  "inserted": 95,
  "skipped": 5,
  "created_categories": [],
  "created_sources": ["New Source"],
  "errors": []
}
```

### Error Responses

```
// Invalid or expired session
{
  "error": "Import session not found or expired. Please re-upload your file."
}

// Session expired
{
  "error": "Import session expired. Please start a new import."
}
```

## Design Decisions

1. **File Caching:** The uploaded file is cached server-side during preview and referenced by an import session ID in the confirmation step. This provides better UX as users don't need to re-upload.

2. **Session-based Import Flow:**
   - Preview stores file with unique session ID (UUID)
   - Files expire after 30 minutes (cleanup via TTL or scheduled task)
   - Confirmation uses session ID to retrieve cached file

3. **Mappings as JSON:** The confirmed mappings are sent as a JSON string in the request body.

4. **Reference Data in Preview:** Categories, sources, and qualifications are included in the preview response so the frontend can show existing options for reference fields.

5. **Backward Compatibility:** The existing endpoint signature changes, but this is acceptable as the frontend will be updated to use the new flow.
